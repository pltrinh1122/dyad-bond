# IT vs IT_iv — the reverse arrow *(Operator riff 2026-06-14; CANDIDATE, n=1)*

> **Provenance:** Operator riff 2026-06-14, *"a separate experiment. does IT produce more reliable
> outcome than IT_iv (intent grounded by invariant)?"* The dual of Phase 1: that grounded an **invariant
> on an intent** (`intent → invariant`); this grounds an **intent by an invariant** (`invariant → intent`).
> Together = the full `I ↔ invariant` cycle (`relationship-craft.md`: "the CYCLE bears the load, not
> either arrow alone"). Corpus: `dialectic/it-iv-corpus.yaml`.

## The arms

- **IT** — raw intent: a value/fiat slogan with **no violation-condition** (the membership-test in
  `invariant-schema.md` classes this as *value/preference/fiat — "not extracted as law."*).
- **IT_iv** — the **same** intent crystallized **by** an invariant: now it carries an oracle
  (`observability` / violation-condition) and a `serves`-edge back to the raw intent.

## Finding (n=1) — **IT is not "less reliable." IT is *inexpressible* in the substrate.**

```
[FAIL] bond:SUMMIT-IT      ✗ scope / prescription / observability : REQUIRED, missing
[PASS] bond:SUMMIT-ITIV
```

The schema requires a violation-condition (`scope` + `prescription` + `observability`) on **every**
record — *including fiat roots*. So a raw intent **cannot be a corpus record at all**: there is no slot
for an intent that has not yet been grounded by an invariant. The moment an intent enters the formal
substrate it must already wear an oracle — i.e., it is already **IT_iv**. The comparison "IT vs IT_iv"
therefore **cannot be hosted inside the substrate**: IT lives in the *wet / prose* layer (the slogan, the
relationship-craft text, the Operator's wetware); only IT_iv lives in the *formal corpus*. They sit on
different instruments — "more reliable" is not a within-substrate measurement. (Same boundary as thread-W's
*git-addressed ≠ wetware-readable*, and the D7 *reachable ≠ valid instrument* rub.)

## The dual — the substrate holds only the cycle's grounded interior

| Raw endpoint | Status in the substrate | Shown by |
|---|---|---|
| **IV** — invariant without intent-grounding | **illegal** (`grounded_in required_iff root==false`) but detection-identical | Phase-1 rig |
| **IT** — intent without invariant-grounding | **inexpressible** (no record without a violation-condition) | this rig |

Both raw endpoints of the `I ↔ invariant` cycle are excluded — one as *illegal*, one as *inexpressible*.
The formal substrate **is the cycle's grounded interior, not its endpoints.** This is why the static
instrument cannot answer either experiment's reliability question: the thing that would make the raw arm
"unreliable" is invisible (Phase 1) or absent (here) by construction.

## The substantive hypothesis — the Operator's "IT more reliable" has real teeth (Goodhart + descent)

Why might raw intent (IT) produce a *more* reliable outcome than crystallized intent (IT_iv)? Two of the
schema's own guards predict exactly this, and only over time:

- **Goodhart** (the schema's `math.use: const breach-detector` guard): IT_iv supplies a measurable oracle.
  The instant that oracle is *pursued* rather than *checked*, it becomes an objective-function proxy that
  is gamed while the wet intent it served drifts away. **Raw IT resists Goodharting precisely because it
  has no gameable oracle.**
- **Descent-guard** (form-ladder): pushing an intent down slogan → tuple → math is lossy; a slogan-form
  intent (SUMMIT is deliberately `form: slogan`) holds richness that the tuple narrows. Premature descent
  trades validity for precision (the D7 lesson, again).

So "IT > IT_iv" is **not** refuted by this rig — it is *out of this rig's reach*. It is a
**consequentialist** claim (`relationship-craft.md`: accuracy = *does enforcing the form produce the world
the intent wants*, tested by breach-behaviour over time), measurable only by **outcome-over-time**.

## Refuted-if / watches

- **The crystallization is pure gain** — *refuted-if* an outcome-over-time rig shows IT_iv tracks the
  intent without drift under pursuit-pressure → then descent is lossless and raw intent buys nothing.
- **Counterfeit-equivalence** — reporting "IT FAILS, IT_iv PASSES" as "IT_iv is more reliable" is the
  inverse counterfeit: IT failed because it is *out-of-domain*, not because it detects worse. Guarded by
  the inexpressibility framing.
- **n=1, hand-built**; SUMMIT chosen because it is a real `slogan`-form fiat intent.

## Next probe (converge-open, no CTA) — the same frontier as Phase-1b
